About the Project

A Tool for Researchers

RCTrials was developed to help researchers gather survey responses from trial participants. It does not gather any personally identifying information (PII) from participants so it is completely double-blinded. It can support multiple groups, pre/during/post trial surveys, and CSV exporting for statistical analysis.

A Tool for Developers

The tool can also be used by app developers to integrate with the RCTrials API and add a randomized controlled trial (RCT) to their mobile app. The users can be divided into multiple groups, each group can be assigned specific app features for A/B testing and surveyed for feedback. Read more about the API here.

Using the Project

Hosted Solution

Use the solution hosted at RCTrials.tk to define your study along with the Surveyor app to collect survey responses.

Integrated

Integrate with the RCTrials API by using the hosted solution and adding the RCT survey functionality to your own app.

Custom Solution

Use any or all of the source code to host your own server, or modify the app to suit the needs of your research.

Hosted Solution

The Hosted Solution allows the researcher to define the parameters of the study - the start and end dates for the registration and trial, how many groups, what features are enabled for each group, and what surveys each group needs to complete. The system generates a four-digit code which the researcher then gives to the trial participants. When the participants submit survey answers the researcher is able to see aggregate information on the survey responses from all participants in each group.

The participants download the free Surveyor app and join the trial with the given four-digit code. The app will ask all participants to fill out the pre-test surveys, and when the trial begins split the participants into the defined groups. During and after the trial other surveys can be requested and will appear in the app for the participants in that group.

Integrated Solution

With the Integrated Solution the researcher still uses the hosted service to define the parameters of the study, however they are then able to integrate their own app with the RCTrials API. The API is used by the Surveyor app, which means the researcher has complete flexibility and control over how the features and surveys are implemented for their study.

Use the Integrated Solution if the research deals with testing features of a mobile app, or testing effects of different interventions delivered via a mobile app. Integrating with the API allows the researcher to use all the online features, while delivering the group assignment, feature assignment, and surveys directly to the mobile app users.

The API exposes endpoints that allow the developer to control when the user registers with the research server and receive their unique identifier, when requests for surveys are sent, and when responses are sent back to the server. This means when the participants are using the researcher's app, they can be placed in different groups, and have different app features enabled or disabled based on their assignment. Later, when they are surveyed, valuable information can be gathered about the feedback from different participants based on their group assignment.

Custom Solution

Because this entire project is Open Source, it is possible to use in custom implementations. The project consists of 3 major parts, which are split into separate GitHub repositories.

The Research Server API, which is responsible for managing the connection with the database, managing researcher and participant accounts, and exposing the API endpoints to the internet. Fork this repo if you wish to manage your own data or modify the API endpoints.

The Surveyor Android App, which allows participants to connect to the Research Server, register into a trial, query for surveys, and submit survey answers. Fork this repo if you wish to create an app that connects to a custom server, or has modified app functionality.

And the WebApp, which is the web front-end built in Angular that allows the administration of the Research Server. Fork this repo if you are working with a custom Research Server or if you wish to modify any of the front-end, reporting, or exporting functionality.
